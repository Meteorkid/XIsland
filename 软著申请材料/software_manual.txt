================================================================================
                            一、软件概述
================================================================================

1.1 软件简介

X Island 是一款 macOS 平台的原生应用程序，采用 Dynamic Island（灵动岛）设计
风格，为用户提供统一的 AI 编程代理监控和管理平台。该软件能够实时监控多种
AI 编程代理（如 Claude Code、Cursor、Codex、OpenCode、Gemini CLI 等）的工作
状态，并提供权限审批、问题回答、计划审核等交互功能。

1.2 开发背景

随着 AI 编程助手的普及，开发者往往同时使用多个 AI 代理进行编程辅助。然而，
这些代理分散在不同的终端和 IDE 中，缺乏统一的监控和管理界面。X Island 应运
而生，旨在解决这一痛点，为开发者提供一个优雅、高效的 AI 代理管理工具。

1.3 技术特点

• macOS 原生应用：使用 Swift 语言开发，充分利用 macOS 系统特性
• Dynamic Island 设计：创新的灵动岛交互模式，不干扰用户工作流
• 多代理支持：兼容 13 种主流 AI 编程代理
• 实时监控：通过 Unix Socket 实现实时状态更新
• 零配置启动：自动检测并配置已安装的 AI 代理

================================================================================
                            三、功能说明
================================================================================

3.1 核心功能

3.1.1 统一仪表盘
    将所有 AI 编程代理的状态集中显示在一个浮动面板中，无论它们运行在哪个
    终端或 IDE 中。用户可以一目了然地查看所有代理的工作状态。

3.1.2 实时状态显示
    通过彩色状态点实时显示代理状态：
    • 蓝色：正在工作
    • 绿色：已完成
    • 橙色：需要输入
    • 红色：出现错误

3.1.3 权限审批
    直接在灵动岛界面中审批或拒绝文件/命令权限请求，无需切换窗口。

3.1.4 问题回答
    从灵动岛界面直接回答 AI 代理提出的问题。

3.1.5 计划审核
    在界面内审核并批准 AI 代理的工作计划。

3.1.6 智能通知
    为会话事件提供 8 位音效通知（可按事件类型配置）。

3.1.7 多会话支持
    每个代理支持多个独立会话，分别跟踪和管理。

3.1.8 窗口跳转
    点击会话可跳转到对应的终端标签页或 IDE 窗口（支持 iTerm2 标签级精度）。

3.2 辅助功能

3.2.1 水平拖拽
    沿屏幕顶部边缘左右拖拽灵动岛位置。

3.2.2 会话标题
    显示首个用户提示作为标题，工作区文件夹作为副标题。

3.2.3 多语言支持
    支持中文、英文、韩文、日文、法文。

3.2.4 配额跟踪
    检查 Kimi、DeepSeek、GLM 等 API 的剩余额度。

3.2.5 外接显示器支持
    灵动岛自动跟随鼠标在显示器间切换。

3.2.6 流式思考显示
    代理推理时显示实时动画思考指示器。

3.2.7 活动日志
    按时间顺序显示所有会话的工具调用记录。

3.2.8 SSH 远程管理
    从灵动岛直接连接和监控远程服务器。

3.2.9 面板尺寸自定义
    可调节面板宽度和最大高度（320-600/700 像素）。

3.2.10 静默时段
    基于时间的自动静音，支持跨午夜时段。

3.2.11 自定义静音规则
    基于正则表达式的静音规则，匹配代理类型、工具名称或工作目录。

3.2.12 Bypass/Auto 模式
    自动批准所有权限、问题和计划审核。

3.2.13 子代理嵌套
    可视化缩进和展开/折叠子代理会话。

3.2.14 VS Code 扩展
    从灵动岛直接跳转到 VS Code 终端标签。

3.3 支持的 AI 代理

X Island 支持以下 AI 编程代理：
• Claude Code（完整支持）
• Cursor（完整支持）
• Codex / OpenAI（完整支持）
• Aider（完整支持）
• OpenCode（完整支持）
• GLM / 智谱（完整支持）
• Kimi / Moonshot（完整支持）
• DeepSeek（完整支持）
• Kiro（完整支持）
• CodeBuddy（完整支持）
• Gemini CLI（基础支持）
• Copilot / VS Code（基础支持）
• Droid / Factory（基础支持）

================================================================================
                            四、安装说明
================================================================================

4.1 下载安装

1. 从 GitHub Releases 页面下载最新的 .dmg 安装包
2. 打开 DMG 文件，将 X Island 拖入 Applications 文件夹
3. 启动 X Island

4.2 Gatekeeper 绕过

由于应用未使用 Apple Developer 证书签名，macOS 会在首次启动时阻止。解决方法：

方法一：终端命令
    xattr -cr /Applications/X\ Island.app

方法二：系统设置
    系统设置 → 隐私与安全性 → 向下滚动 → 在 X Island 警告旁点击"仍要打开"

4.3 从源码构建

前置条件：macOS 14.0+，Swift 5.9+

    git clone https://github.com/Meteorkid/XIsland.git
    cd xisland
    bash Scripts/build.sh
    open ".build/X Island.app"

================================================================================
                            五、使用说明
================================================================================

5.1 首次启动

X Island 在首次启动时会自动检测已安装的 AI 代理，并配置相应的钩子。无需
手动设置。

5.2 基本操作

5.2.1 查看状态
    灵动岛以紧凑药丸形态显示在屏幕顶部。当 AI 代理工作时，会显示状态点。

5.2.2 展开面板
    将鼠标悬停在灵动岛上，面板会展开显示所有活跃会话的详细信息。

5.2.3 审批权限
    当代理请求权限时，灵动岛会显示审批界面，用户可直接点击批准或拒绝。

5.2.4 回答问题
    当代理提出问题时，灵动岛会显示输入框，用户可直接输入答案。

5.2.5 跳转窗口
    点击会话卡片，可跳转到对应的终端或 IDE 窗口。

5.3 设置配置

点击齿轮图标或菜单栏图标可打开设置界面，包含以下选项：

• 语言：自动检测或手动选择（中/英/韩/日/法）
• Bypass 模式：自动批准所有请求
• 自动折叠延迟：面板交互后保持打开的时间
• 已完成会话显示：已完成会话的保留时间
• 面板宽度/高度：可调节面板尺寸
• 音量：音效音量调节
• 静默时段：设置自动静音的时间段
• 静音规则：自定义正则表达式匹配规则

================================================================================
                            六、技术架构
================================================================================

6.1 系统架构

X Island 采用客户端-桥接器架构：

┌─────────────────────────────────────────────────┐
│                  X Island App                    │
│                                                  │
│   NotchWindow (NSPanel)                          │
│   ├── CollapsedPillView (状态点)                 │
│   └── Expanded View                              │
│       ├── SessionListView (会话卡片)             │
│       ├── PermissionApprovalView                 │
│       ├── QuestionAnswerView                     │
│       └── PlanReviewView                         │
│                                                  │
│   SessionManager ← Unix Socket ← di-bridge      │
│   AudioEngine (8-bit 音效合成)                   │
│   ZeroConfigManager (自动配置代理)               │
└─────────────────────────────────────────────────┘

6.2 数据流程

1. AI 代理钩子触发
2. di-bridge 读取 stdin JSON
3. 编码为 DIMessage
4. 通过 Unix Socket 发送
5. SessionManager 接收并更新状态
6. SwiftUI 视图实时响应

================================================================================
                            七、注意事项
================================================================================

7.1 系统要求

• 必须运行 macOS 14.0 或更高版本
• 需要授予辅助功能权限（用于窗口跳转）
• 需要网络权限（用于检查更新和 API 配额查询）

7.2 已知限制

• 未签名应用需要手动绕过 Gatekeeper
• 部分 AI 代理仅支持基础功能
• SSH 远程管理需要配置密钥认证

7.3 故障排除

如遇到问题，可通过以下方式诊断：
1. 检查菜单栏图标是否有错误提示
2. 查看活动日志中的工具调用记录
3. 使用 CLI 命令 xisland diagnostics 获取诊断信息
