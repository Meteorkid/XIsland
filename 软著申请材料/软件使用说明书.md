# XIsland 软件使用说明书

**软件名称：** XIsland（AI编程助手智能岛监控平台）

**版本号：** v1.6.1

---

## 一、软件概述

### 1.1 软件简介

XIsland 是一款 macOS 平台的 AI 编程助手统一监控工具。它采用类似 iPhone 灵动岛（Dynamic Island）的交互设计，在屏幕顶部以悬浮胶囊面板的形式，实时展示所有 AI 编程助手的工作状态。

用户无需在多个终端窗口之间频繁切换，通过 XIsland 即可集中管理 Claude Code、Cursor、Codex、Gemini CLI 等 19 种 AI 编程助手的会话，实现权限审批、问题回答、计划审查等交互操作。

### 1.2 运行环境

- 操作系统：macOS 14.0（Sonoma）及以上
- 芯片架构：Apple Silicon（M1/M2/M3/M4）或 Intel x86_64
- 磁盘空间：约 50MB
- 内存需求：约 100MB

### 1.3 安装方式

**方式一：Homebrew 安装（推荐）**

```bash
brew install meteorkid/x-island/x-island
```

**方式二：源码编译安装**

```bash
git clone https://github.com/Meteorkid/XIsland.git
cd XIsland
swift build -c release
cp -r .build/release/XIsland.app /Applications/
```

## 二、功能说明

### 2.1 灵动岛主面板

XIsland 在屏幕顶部中央显示一个紧凑的胶囊形面板，展示当前所有 AI 编程助手的运行状态：

- **蓝色圆点**：助手正在工作（执行工具调用）
- **绿色圆点**：会话正常完成
- **橙色圆点**：需要用户输入（权限请求或问题）
- **红色圆点**：出现错误

点击胶囊面板可展开完整的会话列表，再次点击收起。

### 2.2 多会话管理

XIsland 支持同时监控多个 AI 编程助手会话。每个会话以卡片形式展示，包含助手类型图标、名称、当前工作目录、运行状态指示、工具调用次数和耗时。

### 2.3 权限审批

当 AI 编程助手需要执行文件操作或命令时，XIsland 在面板中显示权限请求卡片，用户可直接选择"允许"或"拒绝"。

### 2.4 问题回答

当 AI 编程助手提出问题时，XIsland 在面板中显示问题内容和输入框，用户可直接输入答案并提交。

### 2.5 计划审查

当 AI 编程助手生成执行计划时，XIsland 展示计划内容，用户可逐条审查并选择"批准"或"拒绝"。

### 2.6 窗口跳转

点击会话卡片，XIsland 自动跳转到对应的终端窗口或 IDE 编辑器。支持 iTerm2（标签级精确跳转）、Terminal.app、Warp、Alacritty、Kitty、WezTerm、VS Code / Cursor / Trae 系列。

### 2.7 音效提示

内置 11 种 8-bit 风格音效，支持为会话开始/结束、工具调用、权限请求、问题提问、计划审查、错误发生等事件配置不同音效。

### 2.8 配额追踪

支持查询 Kimi、DeepSeek、GLM 的 API 配额余额。

### 2.9 SSH 远程管理

支持通过 SSH 隧道连接和监控远程服务器上的 AI 编程助手会话。

### 2.10 会话导出

支持将会话历史导出为 Markdown 或 JSON 格式。

### 2.11 会话回顾

会话结束后可生成 AI 驱动的会话摘要。

### 2.12 静音规则

支持基于正则表达式的自定义静音规则，可按助手类型、工具名称或工作目录过滤通知。

### 2.13 免打扰时段

支持设置免打扰时段，支持跨午夜时段设置。

### 2.14 主题切换

内置多套主题方案，支持亮色/暗色模式自动跟随系统设置。

### 2.15 多语言支持

支持中文、英文、韩文、日文、法文界面。

### 2.16 面板自定义

可调整面板宽度（320-600px）和最大高度（320-700px）。

## 三、使用方法

### 3.1 首次启动

1. 启动 XIsland 后，应用自动检测系统中已安装的 AI 编程助手
2. 对检测到的每个助手自动配置 Hook
3. 首次启动时请求必要系统权限（辅助功能、通知等）
4. 配置完成后胶囊面板出现在屏幕顶部中央

### 3.2 日常使用

1. 正常使用 AI 编程助手进行编程工作
2. XIsland 自动捕获助手状态并显示在面板中
3. 需要审批权限或回答问题时直接在面板中操作
4. 点击会话卡片跳转到对应终端窗口

### 3.3 偏好设置

点击菜单栏图标打开偏好设置，可配置音效、面板尺寸、主题、语言、静音规则、免打扰时段、自动更新等。

## 四、技术架构

### 4.1 系统架构

XIsland 采用三层架构：

**第一层：Agent Hook** — AI 编程助手的 Hook 脚本在事件发生时调用 di-bridge CLI 工具。

**第二层：di-bridge 桥接层** — 轻量级命令行工具，读取标准输入 JSON，编码为 DIMessage 协议消息，通过 Unix 域套接字发送给主程序。

**第三层：XIsland 主程序** — 通过 SocketServer 接收消息，SessionManager 管理会话状态，SwiftUI 视图层实时响应。

### 4.2 通信协议

系统定义了 16 种消息类型：sessionStart、sessionEnd、toolStart、toolComplete、permissionRequest、permissionResponse、question、questionResponse、planReview、planResponse、statusUpdate、progress、subagentStart、subagentEnd、contextCompact、recap。

### 4.3 支持的 AI 编程助手

完整支持：Claude Code、Codex、Cursor、Trae、OpenCode、Aider、Copilot、CodeBuddy、Kimi、DeepSeek、Kiro、Amp

基础支持：Gemini CLI、Qwen、GLM、Droid、Qoder、Pi、Hermes

## 五、常见问题

**Q1: 启动后看不到胶囊面板？**
检查辅助功能权限：系统设置 → 隐私与安全性 → 辅助功能 → 添加 XIsland。

**Q2: 会话没有自动出现？**
确认 Hook 已配置。首次启动自动配置，也可在偏好设置中手动重新配置。

**Q3: 点击无法跳转？**
确保目标终端应用正在运行。
